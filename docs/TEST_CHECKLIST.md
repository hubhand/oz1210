# 개발 기능 테스트 체크리스트

> 개발 서버 실행 후 실제로 확인할 수 있는 테스트 항목들

## 사전 준비

1. 개발 서버 실행
   ```bash
   pnpm dev
   ```
2. 브라우저에서 `http://localhost:3000` 접속
3. 개발자 도구 열기 (F12) - 콘솔 에러 확인용

---

## 1. 반응형 레이아웃 테스트

### 1.1 데스크톱 레이아웃 (화면 너비 768px 이상)

- [ ] **좌우 분할 레이아웃 확인**

  - 좌측에 관광지 목록이 표시되는지 확인
  - 우측에 네이버 지도가 표시되는지 확인
  - 좌우가 각각 50%씩 차지하는지 확인

- [ ] **좌측 리스트 스크롤 확인**

  - 좌측 리스트 영역을 스크롤할 수 있는지 확인
  - 스크롤이 부드럽게 작동하는지 확인
  - 리스트가 단일 컬럼으로 표시되는지 확인 (데스크톱 분할 레이아웃에서)

- [ ] **우측 지도 확인**

  - 지도가 우측 영역을 꽉 채우는지 확인
  - 지도 높이가 적절한지 확인 (최소 600px)
  - 지도에 마커가 표시되는지 확인

- [ ] **리스트-지도 연동 확인**
  - 리스트 항목을 클릭하면 지도가 해당 위치로 이동하는지 확인
  - 지도 마커를 클릭하면 리스트 항목이 강조 표시되는지 확인

### 1.2 모바일 레이아웃 (화면 너비 768px 미만)

- [ ] **탭 UI 확인**

  - 상단에 "목록" / "지도" 탭이 표시되는지 확인
  - 탭 버튼에 아이콘(List, MapPin)이 표시되는지 확인

- [ ] **탭 전환 확인**

  - "목록" 탭을 클릭하면 관광지 목록이 표시되는지 확인
  - "지도" 탭을 클릭하면 네이버 지도가 표시되는지 확인
  - 탭 전환이 부드럽게 작동하는지 확인

- [ ] **모바일 그리드 레이아웃 확인**

  - 목록 탭에서 관광지 카드가 그리드로 표시되는지 확인 (모바일: 1열, 태블릿: 2열)
  - 카드가 적절한 크기로 표시되는지 확인

- [ ] **모바일 지도 확인**
  - 지도 탭에서 지도가 표시되는지 확인
  - 지도 높이가 적절한지 확인 (400px)
  - 지도에 마커가 표시되는지 확인

### 1.3 반응형 전환 테스트

- [ ] **브라우저 창 크기 조절**
  - 창 크기를 조절하면서 레이아웃이 전환되는지 확인
  - 768px 전후에서 레이아웃이 바뀌는지 확인
  - 전환 시 깜빡임이나 레이아웃 깨짐이 없는지 확인

## 2. 무한 스크롤 테스트

### 2.1 기본 무한 스크롤 동작

- [ ] **초기 로딩 확인**

  - 페이지 로드 시 관광지 목록이 표시되는지 확인
  - 초기에는 12개 항목이 표시되는지 확인

- [ ] **스크롤 다운 확인**

  - 목록을 아래로 스크롤할 때 자동으로 다음 페이지가 로드되는지 확인
  - 하단에 "더 불러오는 중..." 메시지와 스피너가 표시되는지 확인

- [ ] **추가 데이터 로드 확인**

  - 스크롤 시 새로운 관광지 카드가 추가되는지 확인
  - 기존 카드가 사라지지 않고 유지되는지 확인
  - 로딩 중에도 스크롤이 가능한지 확인

- [ ] **로딩 완료 확인**
  - 모든 데이터를 불러온 후 "모든 관광지를 불러왔습니다" 메시지가 표시되는지 확인
  - 더 이상 스크롤해도 추가 로드가 시도되지 않는지 확인

### 2.2 필터와 무한 스크롤 조합

- [ ] **지역 필터 변경**

  - 지역 필터를 변경하면 목록이 초기화되는지 확인
  - 새로운 지역의 관광지가 표시되는지 확인
  - 무한 스크롤이 새로운 필터 조건으로 작동하는지 확인

- [ ] **관광 타입 필터 변경**

  - 관광 타입 필터를 변경하면 목록이 초기화되는지 확인
  - 새로운 타입의 관광지가 표시되는지 확인
  - 무한 스크롤이 정상 작동하는지 확인

- [ ] **정렬 옵션 변경**
  - 정렬 옵션을 변경하면 목록이 초기화되는지 확인
  - 정렬된 순서로 관광지가 표시되는지 확인
  - 무한 스크롤이 정상 작동하는지 확인

### 2.3 검색과 무한 스크롤 조합

- [ ] **키워드 검색**

  - 검색창에 키워드를 입력하고 검색 실행
  - 검색 결과가 표시되는지 확인
  - 검색 결과에 대해 무한 스크롤이 작동하는지 확인

- [ ] **검색 + 필터 조합**

  - 검색 키워드를 입력한 상태에서 필터를 변경
  - 검색 결과가 필터 조건에 맞게 업데이트되는지 확인
  - 무한 스크롤이 정상 작동하는지 확인

- [ ] **검색어 삭제**
  - 검색어를 삭제하면 일반 목록으로 돌아가는지 확인
  - 무한 스크롤이 정상 작동하는지 확인

### 2.4 반려동물 필터와 무한 스크롤 조합

- [ ] **반려동물 필터 활성화**

  - "반려동물 동반 가능" 필터를 활성화
  - 반려동물 가능 관광지만 표시되는지 확인
  - 무한 스크롤이 정상 작동하는지 확인

- [ ] **반려동물 크기 필터**
  - 반려동물 필터 활성화 후 크기별 필터 선택
  - 해당 크기의 관광지만 표시되는지 확인
  - 무한 스크롤이 정상 작동하는지 확인

### 2.5 지도와 무한 스크롤 연동

- [ ] **데스크톱 레이아웃**

  - 무한 스크롤로 추가 데이터를 로드하면 지도에 마커가 추가되는지 확인
  - 새로 로드된 관광지의 마커가 지도에 표시되는지 확인

- [ ] **모바일 레이아웃**
  - 목록 탭에서 무한 스크롤로 데이터 로드
  - 지도 탭으로 전환했을 때 새로 로드된 마커가 표시되는지 확인

## 3. 에러 처리 테스트

### 3.1 초기 로딩 에러

- [ ] **초기 데이터 로드 실패**
  - (개발자 도구에서 네트워크 요청 차단 또는 API 키 제거)
  - 전체 에러 메시지가 표시되는지 확인
  - "관광지 목록을 불러올 수 없습니다" 메시지 확인
  - 재시도 버튼이 작동하는지 확인

### 3.2 무한 스크롤 에러 (부분 에러)

- [ ] **추가 데이터 로드 실패**
  - 초기 데이터는 정상 로드된 상태에서
  - 스크롤하여 추가 데이터 로드 시도
  - 네트워크를 끊거나 API 에러 발생
  - **중요**: 기존 목록은 유지되고, 하단에만 에러 메시지가 표시되는지 확인
  - 에러 메시지: "추가 데이터를 불러오는 중 오류가 발생했습니다"
  - 재시도 버튼이 표시되고 작동하는지 확인

### 3.3 네트워크 에러 구분

- [ ] **네트워크 에러 메시지**
  - 네트워크를 끊고 스크롤 시도
  - "네트워크 연결을 확인해주세요" 메시지가 표시되는지 확인
  - (일반 API 에러와 다른 메시지인지 확인)

### 3.4 에러 복구

- [ ] **재시도 기능**
  - 에러 발생 후 재시도 버튼 클릭
  - 데이터가 정상적으로 로드되는지 확인
  - 에러 메시지가 사라지는지 확인

## 4. 성능 및 UX 테스트

### 4.1 로딩 성능

- [ ] **초기 로딩 속도**

  - 페이지 로드 시 로딩 시간이 적절한지 확인 (3초 이내 권장)
  - 스켈레톤 UI가 표시되는지 확인
  - 필터 영역 스켈레톤이 표시되는지 확인

- [ ] **추가 데이터 로딩 속도**
  - 무한 스크롤로 추가 데이터 로드 시 로딩 시간 확인
  - 로딩 인디케이터가 적절히 표시되는지 확인
  - 로딩 중에도 스크롤이 가능한지 확인

### 4.2 스크롤 성능

- [ ] **부드러운 스크롤**

  - 스크롤이 부드럽게 작동하는지 확인
  - 많은 데이터를 로드해도 성능 저하가 없는지 확인
  - 이미지가 lazy loading으로 로드되는지 확인 (스크롤 시 이미지 로드)

- [ ] **스크롤 위치 유지**
  - 필터 변경 시 스크롤 위치가 상단으로 이동하는지 확인
  - (의도된 동작: 필터 변경 시 초기화)
  - 로딩 중 스크롤 위치가 유지되는지 확인

### 4.3 사용자 경험

- [ ] **로딩 피드백**

  - 로딩 중일 때 명확한 피드백이 제공되는지 확인
  - "더 불러오는 중..." 메시지와 스피너 아이콘이 보이는지 확인
  - 스피너가 회전 애니메이션으로 표시되는지 확인

- [ ] **완료 피드백**
  - 모든 데이터를 불러왔을 때 명확한 메시지가 표시되는지 확인
  - "모든 관광지를 불러왔습니다" 메시지 확인
  - 더 이상 스크롤해도 추가 로드가 시도되지 않는지 확인

### 4.4 성능 최적화 확인

- [ ] **리렌더링 최적화**

  - 필터 변경 시 불필요한 리렌더링이 없는지 확인 (시각적으로 확인)
  - 카드가 부드럽게 업데이트되는지 확인

- [ ] **이미지 최적화**
  - 이미지가 스크롤 시점에 로드되는지 확인 (lazy loading)
  - 이미지 로딩 실패 시 기본 이미지로 대체되는지 확인

## 6. 접근성 테스트

### 6.1 스크린 리더 지원 (시각적 확인)

- [ ] **로딩 상태 안내**

  - 로딩 인디케이터에 적절한 ARIA 속성이 있는지 확인
  - (개발자 도구에서 요소 검사로 `role="status"`, `aria-live` 확인)

- [ ] **목록 구조**

  - 관광지 목록이 `role="list"`로 표시되는지 확인
  - 각 카드가 `role="listitem"`으로 감싸져 있는지 확인

- [ ] **에러 메시지**
  - 에러 메시지에 `role="alert"` 또는 `role="status"`가 있는지 확인

### 6.2 키보드 네비게이션 (선택 사항)

- [ ] **키보드 접근**
  - Tab 키로 모든 인터랙티브 요소에 접근 가능한지 확인
  - Enter 키로 카드 클릭이 가능한지 확인

## 7. 콘솔 에러 확인

### 7.1 개발자 도구 확인

- [ ] **콘솔 에러**

  - 개발자 도구 콘솔에 에러가 없는지 확인
  - 경고 메시지가 없는지 확인
  - React 경고가 없는지 확인

- [ ] **네트워크 탭**
  - API 요청이 정상적으로 전송되는지 확인
  - 응답 상태 코드가 200인지 확인
  - 불필요한 중복 요청이 없는지 확인
  - 에러 응답 시 적절한 상태 코드(500 등)가 반환되는지 확인

## 빠른 확인 체크리스트 (5분 버전)

### 필수 확인 사항

1. **반응형 레이아웃**

   - [ ] 데스크톱: 좌우 분할 레이아웃 확인 (좌측 리스트, 우측 지도)
   - [ ] 모바일: 탭 전환 확인 (목록/지도 탭)
   - [ ] 창 크기 조절 시 레이아웃 전환 확인 (768px 기준)

2. **무한 스크롤 기본 동작**

   - [ ] 목록을 아래로 스크롤하면 자동으로 추가 데이터 로드
   - [ ] 하단에 "더 불러오는 중..." 메시지와 스피너 표시
   - [ ] 모든 데이터 로드 후 "모든 관광지를 불러왔습니다" 메시지 표시
   - [ ] 더 이상 스크롤해도 추가 로드 시도 안 함

3. **필터 + 무한 스크롤**

   - [ ] 지역 필터 변경 → 목록 초기화 및 새 데이터 표시
   - [ ] 관광 타입 필터 변경 → 목록 초기화 및 새 데이터 표시
   - [ ] 필터 변경 후 무한 스크롤 정상 작동

4. **검색 + 무한 스크롤**

   - [ ] 검색창에 키워드 입력 후 검색 실행
   - [ ] 검색 결과 표시 및 개수 확인
   - [ ] 검색 결과에 대해 무한 스크롤 정상 작동

5. **지도 연동**

   - [ ] 리스트 항목 클릭 → 지도가 해당 위치로 이동 및 마커 강조
   - [ ] 지도 마커 클릭 → 리스트 항목 강조 표시 (ring 효과)
   - [ ] 무한 스크롤로 로드된 데이터가 지도에 마커로 추가 표시

6. **에러 처리 (중요: 새로 추가된 기능)**

   - [ ] 초기 로드 실패 시 전체 에러 메시지 표시
   - [ ] 무한 스크롤 중 에러 발생 시 → 기존 목록은 유지, 하단에만 에러 메시지 표시
   - [ ] 에러 메시지에 재시도 버튼 표시 및 작동 확인
   - [ ] 개발자 도구 콘솔에 에러 없음

7. **성능 및 UX**

   - [ ] 이미지가 스크롤 시점에 로드되는지 확인 (lazy loading)
   - [ ] 많은 데이터 로드해도 스크롤이 부드러운지 확인
   - [ ] 로딩 중에도 스크롤 가능한지 확인

## 테스트 시나리오 예시

### 시나리오 1: 기본 사용 흐름

1. 페이지 접속 → 관광지 목록 확인
2. 아래로 스크롤 → 추가 데이터 자동 로드 확인
3. 리스트 항목 클릭 → 지도 이동 확인
4. 지도 마커 클릭 → 리스트 강조 확인

### 시나리오 2: 필터 변경 흐름

1. 지역 필터를 "부산"으로 변경 → 목록 업데이트 확인
2. 아래로 스크롤 → 부산 지역 관광지 추가 로드 확인
3. 관광 타입을 "관광지"로 변경 → 목록 초기화 및 업데이트 확인
4. 무한 스크롤 정상 작동 확인

### 시나리오 3: 검색 흐름

1. 검색창에 "경복궁" 입력 → 검색 실행
2. 검색 결과 표시 확인
3. 아래로 스크롤 → 검색 결과 추가 로드 확인
4. 검색어 삭제 → 일반 목록으로 복귀 확인

### 시나리오 4: 반응형 레이아웃 전환

1. 데스크톱 크기에서 좌우 분할 레이아웃 확인
2. 브라우저 창 크기 축소 → 모바일 레이아웃으로 전환 확인
3. 탭 전환 (목록 ↔ 지도) 확인
4. 창 크기 확대 → 데스크톱 레이아웃으로 복귀 확인
